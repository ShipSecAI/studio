## Codebase Patterns
- **NestJS Config Pattern**: Use `registerAs()` in config/*.config.ts files, load in ConfigModule.forRoot() in app.module.ts
- **Global Services**: Services exported from AppModule become globally available to all modules
- **Environment Variables**: Use ConfigService.get() to access config, support optional features with null checks
- **Analytics Directory**: `backend/src/analytics/` already exists for PostHog (product analytics), separate from OpenSearch (security analytics)
- **Graceful Degradation**: Services should log warnings and disable features when dependencies aren't configured (not throw errors)
- **Pre-existing Type Errors**: Some existing type errors in the codebase related to workspace packages not being built - these are pre-existing and can be ignored if they don't relate to your changes
- **Worker Architecture**: Worker is separate from backend and can have its own dependencies (OpenSearch, etc.)
- **ExecutionContext Extension**: To pass workflow-level data to components, extend ExecutionContext in packages/component-sdk/src/types.ts and CreateContextOptions in context.ts
- **Workflow Context Flow**: workflow-runner.ts receives workflowId/organizationId from options → passes to createExecutionContext() → available in component execute() via context
- **Dynamic Imports in Components**: Use `await import('...')` in execute() to avoid circular dependencies and module initialization issues
